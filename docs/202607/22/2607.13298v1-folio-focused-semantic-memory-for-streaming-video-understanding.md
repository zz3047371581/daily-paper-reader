## 摘要
在线流式视频理解中，视频流持续到达，查询可能随时发出。由于流式帧无界增长，系统必须持续压缩和保留从已观察的视频前缀中获取的信息，而未来的帧和未来的查询仍是未知的。核心挑战在于决定保留哪些信息以及如何组织维护的历史记录：随着历史记录随流增长，记忆成本增加，许多冗余的视觉细节被保留，而后续的查询往往依赖于特定的实体、动作及其时序变化。为了解决这一挑战，我们引入了FOLIO，一个无需训练的聚焦语义记忆系统，它能够以更高细节记录流中的重要部分，同时保持周围上下文的紧凑。随着流的到达，FOLIO在动态焦点状态的引导下，在片段级别更新记忆，结合了短期视觉缓冲区和围绕观察到的实体组织并链接到视觉证据缓存的长时语义记忆。在查询时，轻量级混合检索将对结构化记忆的直接匹配与语义查询扩展相结合。FOLIO实现了最先进的性能，在使用Qwen3-VL-8B的OVO-Bench上达到82.0/69.1的感知/后向准确率，在StreamingBench上达到74.5的总体准确率，同时通过为焦点实体保留详细记录并紧凑存储周围上下文，大幅降低了维护流式记忆的成本。

## Abstract
In online streaming video understanding, a video stream continues to arrive and queries may be issued at any time. Because streaming frames grow without bound, the system must continuously compress and retain information from the observed video prefix while future frames and future queries remain unknown. The core challenge is deciding what information to retain and how to organize the maintained history: as this history grows with the stream, memory cost increases and many redundant visual details are retained, whereas later queries often depend on specific entities, actions, and their temporal changes. To address this challenge, we introduce FOLIO, a training-free focused semantic memory system that records important parts of the stream in higher detail while keeping surrounding context compact. As the stream arrives, FOLIO updates memory at the segment level, guided by a dynamic focus state, combining a short-term visual buffer with a long-term semantic memory organized around observed entities and linked to a visual-evidence cache. At query time, lightweight hybrid retrieval combines direct matching over the structured memory with semantic query expansion. FOLIO achieves state-of-the-art performance, reaching 82.0/69.1 Perception/Backward accuracy on OVO-Bench with Qwen3-VL-8B and 74.5 overall accuracy on StreamingBench, while substantially reducing the cost of maintaining streaming memory by reserving detailed records for focused entities and storing surrounding context compactly.

### 2. 论文提出的方法论

FOLIO 的核心是一个无需训练的、包含动态焦点状态的混合语义记忆系统，其工作流程分为在线记忆构建和查询时检索两个阶段。

-   **核心思想**：以实体为中心构建长时记忆，并由一个**动态焦点状态**来引导记忆写入的粒度：焦点实体写入详细状态、位置、关系等信息，非焦点上下文则紧凑化处理。
-   **关键技术细节与系统组件**：
    -   **在线记忆构建循环**：系统按固定时长的视频片段进行流水线式处理。每个片段到来后进行关键帧选取，然后根据前一片段结束后的焦点状态决定当前片段的写入层级，再由VLM生成结构化记录，最后合并到持久化记忆链中。
    -   **动态焦点状态**：核心函数为 `pi(o) = clip[0,1] (γp(i−1)(o) + α⊤ϕ+_i(o) − β⊤ϕ−_i(o))`。其中，`pi(o)` 代表实体 `o` 的焦点分数。正向特征 `ϕ+` 包括实体的可见性、状态/位置变化、参与事件以及**多轮交互的相关性**；负向特征 `ϕ−` 包括实体消失和静态背景行为。分数通过加权方式更新，并用于指导下一片段的写入层级。多轮查询的术语会提高相关实体的焦点分数。
    -   **混合记忆结构**：系统的最终记忆状态 `Mi` 由三部分组成：
        1.  **短期视觉缓冲区**：存放最近的视频帧，为当前状态提供高保真视觉证据。
        2.  **长时语义记忆**：以**实体分类账**为中心组织的结构化记忆。每个实体条目包含规范名称、别名、类别，并随时间累积其状态、位置、关系、动作链等信息，并链接到视觉证据缓存。
        3.  **视觉证据缓存**：存储用于构建长时记忆的关键帧，以各后续追溯。
    -   **聚焦引导的在线记忆写入**：
        -   记忆写入按片段触发。系统首先进行**自适应关键帧选择**，选取边界帧、中间帧和视觉变化最大的帧。
        -   **写入器VLM** 接收所选关键帧和由焦点状态导出的写入层级，产出结构化记录。对于焦点实体，记录其精细的状态、位置变化和关系；对于非焦点实体，仅记录紧凑的上下文笔记。
        -   新的记录通过基于规则的姓名、别名和类别匹配，与现有长时记忆进行合并。匹配成功的追加到对应实体的记忆链，不成功的则创建新的实体条目，从而避免相似但不同实体的错误合并。
    -   **轻量级混合检与回答**：
        -   **查询解析**：解析自然语言查询，提取目标、锚点、动词、时间范围等。
        -   **直接匹配**：将查询和选项中的实体、动作名称与长时记忆中的结构化字段进行匹配，并排序打分。
        -   **语义查询扩展**：当直接匹配不成功时，触发一个轻量级VLM调用，将抽象的查询概念（如"reader"）映射到记忆库中已有的实体（如"book"），以检索到相关记录。
        -   **证据组装与回答**：将匹配到的结构化记忆、短期缓冲区和从缓存中恢复的相关关键帧共同组装为证据块，提交给**回答器VLM**，生成最终答案。

### 8. 不足与局限

-   **固定的时间片段划分**：系统使用固定的8秒片段进行记忆写入，论文自己也指出，对于某些场景，基于事件触发（如镜头切换、动作终止）的自适应片段划分可能会更有效。
-   **通用记忆模式的能力边界**：写入器VLM使用一个通用的记忆框架（状态、动作、关系等）。对于特定领域非常专业或细微的线索（如球类比赛中的得分变化、物品归属权的转移），可能需要更特化的记忆字段才能被有效捕捉。
-   **写入器的质量瓶颈**：系统的记忆质量高度依赖于写入器VLM的性能。案例分析和失败分类显示，主要误来源于写入器没有提取到足够精准的信息（记忆缺失）或提取的信息不够有区分度，而不是检索失败。这指出**提升记忆构建的精度**是进一步发展的关键。
-   **幻觉检测任务的高度依赖性**：论文提到了在幻觉检测任务上性能的显著提升，但也坦承这部分提升受益于“保守的回答校准”策略，不完全归功于记忆系统本身的质量。
