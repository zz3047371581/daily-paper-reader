## 摘要
使用多模态大语言模型（MLLM）理解长视频需要从数千个候选帧中选出一组紧凑的帧，但识别正确的帧似乎需要先理解视频。我们通过一个简单的观察解决了这一循环依赖：在验证选定的提取层上，MLLM中的跨模态注意力已经提供了与查询相关的帧证据，而无需自回归生成。我们利用这一特性构建了DAFS（基于动态注意力的预算感知帧选择），一种无需训练的帧选择器。轻量级MLLM选择器，即使只有20亿参数，也能通过查询条件聚合将选定层的注意力转换为相关性分数，从而提取帧级证据。这无需自回归解码即可实现跨帧比较。为了应对选择器自身的上下文限制，我们将候选池大小和每帧令牌预算的联合分配形式化为一个离散优化问题，并通过动态规划求解。在32帧预算下，我们的选择器在Video-MME上比均匀采样最多提高6.4个百分点，并在匹配帧预算下优于先前的基于训练的选择器，同时在不同选择器和回答器架构以及不同任务中泛化，无需重新训练。

## Abstract
Understanding long videos with multimodal large language models (MLLMs) requires selecting a compact set of frames from thousands of candidates, yet identifying the right frames seemingly requires understanding the video first. We resolve this circular dependency with a simple observation: cross-modal attention at validation-selected extraction layers in MLLMs already provides query-relevant frame evidence without requiring autoregressive generation. We exploit this property to build DAFS (Dynamic Attention-based Budget-aware Frame Selection), a training-free frame selector. A lightweight MLLM selector, even with only 2B parameters, can extract frame-level evidence by converting selected-layer attention into relevance scores through query-conditioned aggregation. This enables cross-frame comparison without autoregressive decoding. To handle the selector's own context constraint, we formulate the joint allocation of candidate pool size and per-frame token budget as a discrete optimization problem solved by dynamic programming. Under a 32-frame budget, our selector improves over uniform sampling by up to 6.4 points on Video-MME and outperforms prior training-based selectors under matched frame budgets, while generalizing across selector and answerer backbones, and across tasks, without retraining.

- 现有方案存在不足：均匀采样内容盲；CLIP 类方法缺乏跨帧上下文和强查询条件；MLLM 类自回归选择器需额外训练且推理成本高。
- 本文的核心含义是：可以利用轻量 MLLM 内部特定层的跨模态注意力信号，在无需自回归生成、无需训练的框架下，实现高效的查询感知帧选择，并解决选择器自身的上下文预算问题。

## 2. 方法论
### 总体框架
- 提出 **DAFS（Dynamic Attention-based Budget-aware Frame Selection）**，一个无训练的轻量级帧选择器。
- 流程四步：① 以 1 FPS 抽帧并按视频时长均匀下采样构建候选池；② 动态池化压缩每帧视觉 token，保证总 token 数不超过全局预算 B；③ 通过查询条件聚合从选定层的跨模态注意力中计算帧级相关性分数；④ 按分数排序选取 top-K 帧送入固定的“回答器” MLLM 进行推理。

### 关键技术细节
- **注意力层选择**：通过验证集（Charades-STA 和 NExT-GQA）扫表找出中层交叉注意力层（如 InternVL3.5‑2B 的 L12），该层注意力集中在查询相关视觉 token 上，且尚未被深层融合模糊。
- **查询感知帧评分**：
  - 提取选定层 ℓ⋆ 的文本到视觉注意力矩阵，平均多头并归一化。
  - 对每一帧 j，汇聚该帧内所有视觉 token 上的注意力权重得到 `S_{t,j}`。
  - 定义“重要 token 集合” `T_imp`（多项选择时包含选项 token，开放问答用 LLM 抽取实体 token），采用 max pooling：`φ_j = max_{t∈T_imp} S_{t,j}`，得到帧级显著性分数。
- **预算感知的帧采样与 token 压缩**：
  - 按视频时长 d 稀疏采样 `F(d)` 个候选帧。
  - 每帧通过非重叠平均池化压缩为 `P(d)` 个视觉 token，`P(d)` 从 {16,25,36,49,64,100,144} 中选取。
  - 联合约束 `F(d) * P(d) ≤ B`（B=32,000）。
- **动态规划（DP）校准分配策略**：
  - 将时长离散化为有序桶 `d1<d2< ... <dN`，强制单调约束：`F_i ≤ F_{i+1}`，`P_i ≥ P_{i+1}`。
  - 目标是在校准开发集上最大化 QA 准确率之和，通过 DP 求出**不随时间变短的分辨率，不减少的帧数**的最优 (`F_i`, `P_i`) 序列。
  - 推理时 O(1) 查表得到当前视频的 (`F`, `P`) 配置。

### 公式要点
- `φ_j = max_{t∈T_imp} Σ_{p=1}^{P} A^{(ℓ⋆)}_{t,(j,p)}`
- DP 目标：`max Σ Acc_i(F_i,P_i)` 满足 `F_i·P_i ≤ B`, `F_i ≤ F_{i+1}`, `P_i ≥ P_{i+1}`

## 6. 主要结论与发现
- 中层交叉注意力天然编码了查询相关帧的证据，可在无训练条件下转化为有效的帧选择信号。
- DAFS 在 32 帧预算下，视频理解准确率显著优于均匀采样、CLIP 类和部分训练型 MLLM 选择器（Video-MME +6.4 点，LVB +5.5，MLVU +11.5）。
- 方法轻量：选择步骤仅约 0.47 秒，总延迟低于训练型方案，显存占用更小。
- 选择器与答案器解耦，选择的帧可提升多种答案器性能，且可推广到工具使用工作流。
- 动态规划分配可有效权衡时间覆盖与空间细节，优于任何静态 (F,P) 配置。

## 7. 优点
- **无需训练**：完全基于冻结预训练 MLLM 的注意力信号，不更新模型参数，仅需推理校准。
- **高效解耦**：选择器和答案器分离，可灵活替换更强答案器，无需重新训练即可获得增益。
- **轻量快速**：选择阶段仅一次前向 + 注意力聚合，显著快于自回归选择方法。
- **预算感知**：DP 联合优化候选帧数和每帧 token 数，自动适应不同时长，兼顾分辨率与覆盖度。
- **实验全面**：涵盖多个基准、不同预算、跨模型泛化、跨任务、消融与效率评估，论证扎实。

## 8. 不足与局限
- **依赖预训练 MLLM 的注意力质量**：若基础 MLLM 本身注意力不聚集在关键证据上，性能可能下降；需要针对不同选择器手动校准提取层。
- **校准需要外部数据**：虽然验证集与评测集无重叠，但确定提取层 ℓ⋆ 和 DP 分配仍需额外的时序定位/QA 数据，不同应用可能需要重新校准。
- **重要 token 集合的定义**：任务相关，对开放式问题依赖 LLM 抽取，可能引入误差。
- **仅测试固定数量的新近基准**：未在更长时间视频（小时级）或更细粒度时序任务（如 dense captioning）上验证。
- **选择器架构较小**：主实验最多用到 8B 的选择器，未探索更大选择器可能带来的收益或成本。
- **压缩信息损失**：选择端对每帧做强压缩（最低 16 tokens），极端压缩可能丢失细节，尽管最终答案器使用原帧。
