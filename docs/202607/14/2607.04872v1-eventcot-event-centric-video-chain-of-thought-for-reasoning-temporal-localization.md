## 摘要
推理时序定位（RTL）要求模型生成一个本身包含支持时间间隔的答案，因此必须在单一响应中联合产生高层推理和精确的时序定位。为了解决这一挑战性任务，我们提出了首个以事件为中心的视频思维链框架，称为EventCoT。EventCoT首先对输入视频进行以事件为中心的标记化，将其转换为紧凑的事件标记，从而能够高效识别与问题相关的事件。然后在识别出的事件内进行推理以生成答案，通过占位符标记与视觉嵌入的对齐嵌入匹配来定位时间间隔。EventCoT在ActivityNet-RTL上的推理时序定位任务中达到了最先进的成果，同时使用的视觉标记数量远少于此前工作。为了验证其通用性能，我们进一步在基于定位的视频问答基准ReXTime上评估了EventCoT，并获得了强大的零样本结果。

## Abstract
Reasoning temporal localization (RTL) requires a model to generate an answer that itself contains the time interval supporting it, so high-level reasoning and precise temporal grounding must be produced jointly in a single response. To tackle this challenging task, we propose the first event-centric video chain-of-thought framework, dubbed EventCoT. EventCoT first performs event-centric tokenization of the input video to convert it into compact event tokens, enabling efficient identification of question-relevant events. It then reasons within the identified events to generate the answer, grounding the time interval via embedding matching that aligns placeholder tokens with visual embeddings. EventCoT achieves state-of-the-art results on ActivityNet-RTL for reasoning temporal localization while using substantially fewer visual tokens than previous work. To verify its general performance, we further evaluate EventCoT on the grounded video question answering benchmark ReXTime, where it attains strong zero-shot results.

- **任务背景**：推理时序定位（Reasoning Temporal Localization, RTL）要求模型在回答一个开放式问题时，同时输出一个支持答案的**具体时间区间**（开始‑结束时间戳）。这和传统的视频问答（时间边界最多作为辅助输出）不同，**推理和精确定位必须在同一个响应中联合完成**，极具挑战性。
- **现有方法的问题**：
  - 基于均匀帧采样的视频‑大语言模型（如LITA）会引入大量**冗余视觉令牌**，且采样密度难以平衡；对长视频尤其低效。
  - 时间定位往往采用**离散时间令牌的下一个令牌预测**，这**将时序定位与答案生成纠缠在一起**，既降低了定位精度，也拖慢了推理速度。
  - 近期的工作虽然尝试迭代检索与问题相关的视频片段，但需要**多次调用LLM、处理空间细节特征，导致计算代价高、令牌用量大**，并且没有从根本上解开推理与定位的耦合。
- **论文的整体含义**：提出一种**以事件为中心的视频思维链框架EventCoT**，通过将视频转化为紧凑的“事件”序列，先粗选与问题相关的事件区域，再在选定区域内精细推理并预测时间边界，从根本上解耦推理和定位，实现**更少视觉令牌、更高定位精度和更好回答质量**。

## 2. 方法论

### 2.1 核心思想
- 模拟人类认知中**将连续活动理解为离散事件序列**的特性，把视频分割为**非重叠的语义事件**。
- 采用**两阶段链式思考（Chain‑of‑Thought）**：
  1. **第一步：问题相关事件选择**——利用紧凑的“事件令牌”快速过滤出与问题相关的视频区间。
  2. **第二步：精细推理与时间定位**——在选中的事件对应的帧特征上，通过**嵌入匹配**（而非离散令牌预测）预测开始/结束时间戳并生成文本答案。

### 2.2 关键技术细节与流程

#### 2.2.1 以事件为中心的标记化（Event-centric Tokenization）
1. **帧特征提取**：从视频中均匀采样T帧，用冻结的CLIP‑L/14编码，经投影层和空间池化得到帧特征\(\mathbf{F} \in \mathbb{R}^{T \times D}\)，并加入旋转位置嵌入。
2. **事件边界检测**：
   - 采用改进的**密度峰值聚类与K近邻（DPC‑KNN）**。
   - 距离定义结合特征距离与时间邻近性：\(d(i,j) = (1-\alpha)\|f_i - f_j\|_2 + \alpha \frac{|i-j|}{T}\)。
   - 局部边界评分 \(\rho_i\) 取K近邻平均距离；距离因子 \(\delta_i\) 衡量帧的独特性，并通过**幂缩放（power scaling）**放大高峰、抑制噪声。
   - 最终边界评分 \(b_i = \rho_i \cdot \tilde{\delta}_i\)，选取前N‑1个最高分帧作为边界，形成N个事件。
3. **每事件嵌入（事件令牌）**：
   - 先对每个事件内帧特征做平均池化，得到事件点特征。
   - 再通过**事件上下文注意力**，允许事件点特征关注自身及前后各β帧内的帧特征，输出**事件令牌**\(\mathbf{e}_n\)。
   - 引入注意力掩码以保持局部性，提高边界鲁棒性。

#### 2.2.2 第一步：问题相关事件选择
- 在LLM词汇中引入占位符`<segment_key>`。
- LLM处理指令、问题和事件令牌后，产生该占位符的隐藏嵌入\(\mathbf{h}_{sk}\)。
- 对每个事件令牌的LLM最终隐藏嵌入\(\mathbf{h}_{e_n}\)计算相关性分数：
  \[
  r_n = \sigma\left(\frac{\mathbf{h}_{sk}^\top \mathbf{h}_{e_n}}{\tau}\right)
  \]
- 选择 \(r_n > \gamma \cdot \max_n r_n\) 的事件作为相关事件。

#### 2.2.3 第二步：精细推理与时间定位
- 从选定事件的帧中构建帧级视觉令牌；同时随机混入30%的未选中事件的帧，打乱真实事件边界的暗示。
- LLM自回归生成答案，其中包含占位符`<start>`和`<end>`。
- 定位过程：利用占位符的最终隐藏嵌入\(\mathbf{h}_{ts}\)（或\(\mathbf{h}_{te}\)）与当前可见的帧特征计算相似度：
  \[
  p_i = \text{softmax}\left(\frac{\mathbf{h}_{ts}^\top \mathbf{h}_{f_i}}{\tau}\right)
  \]
- 选最高相似度的帧索引转换为时间戳，替换占位符，形成最终答案。

#### 2.2.4 训练目标
- **语言建模损失**：标准的自回归交叉熵损失。
- **事件选择损失**：基于与真实时间区间的IoU（重叠至少1帧为正）的二元交叉熵损失。
- **时序定位损失**：
  - 以高斯平滑标签的交叉熵损失（目标分布 \(g_i = \exp(-(t_i - t_{gt})^2 / 2\sigma^2)\)）。
  - DIoU损失：直接回归预测区间与真实区间的距离交并比损失。
- 总损失：\( \mathcal{L} = \mathcal{L}_{lm} + \lambda_{sel} \mathcal{L}_{sel} + \lambda_{ts} \mathcal{L}_{ts} \)

- EventCoT在ActivityNet‑RTL上同时实现了**最佳的时序定位（mIoU 58.5, P@0.5 66.8）和答案质量（GPT‑4评分48.7）**，而使用的视觉令牌数量仅为LITA的约16%（平均39 vs. 356）。
- 在ReXTime上，EventCoT展现出**强大的零样本泛化能力**，即使采用较弱的Vicuna‑7B骨干，定位指标也超越以往方法，采用Qwen2.5‑7B后全面最优。
- **事件为中心的标记化**比均匀切分更能准确捕捉语义边界，并显著提升定位质量。
- **嵌入匹配**比离散时间令牌预测更有利于解耦推理与定位，获得更高精度和更高吞吐量。
- 两阶段粗-精过程相互协同：精准的事件选择能极大提升第二阶段的定位精度；即使选择略有偏差，引入的随机背景事件和帧级匹配也对性能有一定补救作用。

## 7. 优点（方法或实验设计的亮点）

- **问题建模清晰且高效**：首次将视频事件作为CoT中间步骤，把复杂RTL任务分解为粗选和精调，直观且符合人类事件感知。
- **大幅减少视觉令牌**：事件令牌极其紧凑，显著降低LLM推理的计算量和延迟，提高实用性。
- **解耦推理与定位**：通过嵌入匹配分别独立优化事件选择和时序边界预测，避免多目标冲突。
- **事件边界检测效果优异且轻量**：结合时间邻近与幂缩放的DPC‑KNN，无需任何学习即可获得高边界召回（94.3%），远优于传统镜头检测方法，并保持极低的时间和内存开销。
- **训练策略巧妙**：在第二阶段混入随机未选中事件，成功打破边界泄漏的捷径，迫使模型学习真正的时序推理。
- **实验严谨全面**：涵盖多种基线、两个基准、多维度消融，并分析了特征判别性、推理时间和各阶段相互影响，结论可信度高。

## 8. 不足与局限

- **固定事件数和采样帧数**：当前均为T=100帧、N=16事件，专为ActivityNet等约数分钟的视频设计，**难以直接适应超长视频或极短视频**。未来需自适应调整。
- **定位与生成耦合仍存**：占位符嵌入需同时承担语言建模和匹配两个角色，尚未完全分离，可能限制极致性能。文中指出潜在解决方案为“潜在思维链（latent CoT）”。
- **缺少空间细节**：采用空间池化后的帧特征，对于需要精细空间定位的查询（如“这个人挥手时在哪一帧？”的精确位置）支持不足。
- **事件边界检测依赖启发式**：虽然效果好，但边界检测基于手工设计的距离和阈值，可能在复杂场景（如渐变过渡）中失效。
- **训练数据依赖**：尽管零样本效果不错，但完整的RTL训练依赖于含时间标注的数据集，构造此类数据成本较高。
- **骨干网络扩展性**：实验仅限于7B参数级别，未探索更大模型下的表现，不过文中已有更换骨干预实验，趋势良好。
