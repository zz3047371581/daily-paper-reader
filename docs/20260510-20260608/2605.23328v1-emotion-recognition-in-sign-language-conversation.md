## 摘要
对话情绪识别是情感计算的核心组成部分，而当前的手语情绪数据集主要关注孤立语句，缺乏对话上下文。仅在孤立话语上训练的模型在真实场景中性能下降，因为它们无法利用历史对话流。为了解决这一结构局限性，我们将ERC任务引入手语视频分析，并提出了eJSL Dialog数据集。该数据集使用STUDIES语料库的脚本构建，包含1920个视频样本，组织成480段独特对话。我们使用从孤立视觉网络到多模态对话架构的模型在该数据集上进行了系统基准测试。结果表明，将通用多模态对话情绪识别模型应用于手语时存在领域差距。这些发现表明，明确需要专门针对手语的上下文感知视觉提取器，并指出扩展对话数据集的规模以支持大规模预训练是未来研究的必要步骤。

## Abstract
Emotion Recognition in Conversation is a core component of affective computing, while current resources of sign language emotion datasets primarily focus on isolated sentences and lack conversational context. Models trained exclusively on these isolated utterances demonstrate degraded performance in real world scenarios because they cannot utilize historical dialogue flow. To address this structural limitation, we introduce the ERC task to sign language video analysis and propose the eJSL Dialog dataset. Constructed using the scripts from the STUDIES corpus, the dataset contains 1,920 video samples organized into 480 unique dialogues. We conduct systematic benchmarking on this dataset using models ranging from isolated visual networks to multimodal conversational architectures. The results reveal a domain gap when applying generic multimodal conversational emotion recognition models to sign language. These findings demonstrate the explicit need for context aware visual extractors specific to sign language and indicate that expanding the scale of conversational datasets to support large scale pre-training is a necessary next step for future research.

- **数据集与场景**：
    - **基准测试集**：本研究提出的 **eJSL Dialog** 数据集，包含 480 段对话，共 1920 个视频片段。该数据集仅用作评估和测试的“黄金标准”。
    - **预训练集**：由于基线视觉模型需要基本的视觉表征，使用了 **BOBSL**（英国手语数据集）进行预训练，并将 BOBSL 标签映射为与本任务对齐的四类情绪。
- **对比方法**：模型覆盖了三个维度，共五个基线：

    - **纯文本模型（有上下文）**：
        - **EmoTrans**：基于文本输入的、利用情感状态转变进行对话情绪识别的模型。

        - **MMGCN**：利用说话者信息，通过图卷积网络建模说话者内部和之间的多模态依赖关系。
- **评估指标**：主要使用加权 F1 分数。

### 6. 主要结论与发现

- **领域鸿沟的存在**：通用多模态对话情绪识别模型（如 TelME, MMGCN）在手语任务中性能严重下降（加权 F1 分别仅为 10.46% 和 8.5%），远低于文本模型。这表明为非手语者设计的视觉特征提取器在手语中会引入噪声，干扰了文本模态的有效信息。
- **文本信息的优势与局限**：基于文本的对话模型 EmoTrans 取得了最高的整体性能（55.40% F1），证明了脚本的语义与情绪标签高度相关。然而案例分析也指出，当文本不含明显情绪词时，文本模型会失败，此时视觉线索至关重要。
- **视觉信息的价值与挑战**：结合手部和面部线索的视觉模型 EANwH 优于仅用面部的 EmoAffectNet，证实了手语视觉特征包含独立的情感信息。但其对“悲伤”类别的识别能力很弱。
- **未来方向**：明确需要开发专门针对手语、且能感知对话上下文的视觉特征提取器，并构建更大规模的对话数据集以支持大规模预训练。

### 7. 优点

- **问题新颖，研究动机强**：首次将 ERC 任务引入了手语视频分析领域，精确地指出了现有研究“忽视上下文”的结构性缺陷，具有很高的应用价值和社会意义。
- **数据集填补空白**：构建并清晰地形式化了首个手语多轮对话情绪识别基准数据集（eJSL Dialog），为未来研究提供了宝贵的标准化评估平台。数据集结构清晰，情绪标签平衡。
- **基准评估具有洞见**：通过覆盖不同模型范式的系统性基准测试，清晰地揭示了通用多模态模型在手语上的“领域鸿沟”，为未来模型设计指明了方向。
- **分析深入**：案例分析部分直观地展示了上下文、文本语义和视觉线索在情绪理解中的互补作用，使抽象结论更具说服力。

### 8. 不足与局限

- **数据集规模与生态效度**：数据集仅由两名聋哑演员在实验室环境下录制，背景单一，对话由脚本设定而非自然发生。模型泛化到“真实世界”复杂环境（不同背景、光照、大量用户）的能力存疑。
- **模态信息有限**：仅提供 2D RGB 视频，缺少深度信息或精细的面部/手部网格数据，可能限制了模型捕捉微妙表情和手势的能力。
- **缺乏关键消融实验**：没有对多模态模型的失败进行定量分析，例如，控制或替换视觉特征后，观察模型性能是否能恢复，以精确诊断问题源头。
- **文本翻译偏差**：将日文翻译成英文以适配模型，可能会引入翻译误差或丢失文化细微差别，影响实验结果的精确性。
- **算力信息缺失**：没有报告模型训练的计算成本，使得其他研究者难以评估实验的可复现性和资源门槛。
