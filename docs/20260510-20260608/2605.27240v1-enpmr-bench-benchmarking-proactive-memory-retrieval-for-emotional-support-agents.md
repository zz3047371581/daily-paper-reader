## 摘要
记忆增强的语言代理正越来越多地部署于情感支持等情感应用中，其中理解并响应用户的潜在情感需求至关重要。然而，现有研究通常将记忆视为事实检索的工具，忽略了其在塑造用户情感体验方面的作用。在本工作中，我们引入了ENPMR-Bench，一个用于评估情感需求感知的主动记忆检索（ENPMR）能力的基准，该能力是使代理能够推断用户潜在情感需求并主动检索适当记忆以支持共情交互的核心能力。ENPMR-Bench以马斯洛需求层次理论为基础，包含超过1800个记忆增强对话，并定义了情感需求与支持性记忆类型之间的结构化映射。实验结果表明，当前的检索范式，包括基于嵌入和大语言模型驱动的方法，均存在显著缺陷，其共情得分明显低于黄金记忆条件。虽然思维链提示在一定程度上改善了推断情感需求与检索记忆之间的一致性，但仍有明显的性能差距。总之，这些发现揭示了当前代理的关键局限，并指出了通过需求敏感的主动记忆检索推进个性化情感支持的方向。

## Abstract
Memory-augmented language agents are increasingly deployed in affective applications such as emotional support, where understanding and responding to users' latent emotional needs is critical. However, existing research often treats memory as a tool for factual retrieval, overlooking its role in shaping users' emotional experiences. In this work, we introduce ENPMR-Bench, a benchmark for evaluating Emotional Need-aware Proactive Memory Retrieval (ENPMR), a core capability that enables agents to infer users' latent emotional needs and proactively retrieve appropriate memories to support empathetic interaction. Grounded in Maslow's hierarchy of needs, ENPMR-Bench includes over 1,800 memory-augmented dialogues and defines structured mappings between emotional needs and supportive memory types. Experimental results demonstrate that current retrieval paradigms, including both embedding-based and LLM-driven approaches, exhibit substantial deficiencies, with empathy scores significantly lagging behind golden memory conditions. While chain-of-thought prompting improves the alignment between inferred emotional needs and retrieved memories to some extent, a notable performance gap remains. Together, these findings reveal critical limitations in current agents and outline directions for advancing personalized emotional support through need-sensitive memory retrieval.

## 3. 实验设计

### 3.1 基准构成与数据量

- **记忆库**：共 11,846 条记忆，涵盖高光（3,729）、能力（3,463）、关系（2,324）、目标（1,719）、偏好（611）。

### 3.2 评估任务与对比方法

任务分为两个环节：

- **任务1：记忆检索**
  - **评估模型**：对比多款中文强大的嵌入模型：**gte-Qwen2-7B-instruct**、**Qwen3-Embedding-8B**、**doubao-embedding**、**text-embedding-3-large**。
  - **指标**：Recall@k、Precision@k、nDCG@k（\(k\in\{1,3,5,10\}\)）。

- **任务2：回复生成**
  - **LLM 评估设置**：对比 **Qwen-Max**、**DeepSeek-V3**、**GPT-4o**、**Gemini-2.5-Flash** 四个模型。每个模型在三种设定下评估：
    - `base`：模型自主选记忆。
    - `w/CoT`：带思维链推理的记忆选择与回复生成。
    - `w/Golden`：直接给定 ENPMR-Bench 标注的黄金记忆。
  - **Agentic Memory 评估**：对比当前 SOTA 记忆系统 **Mem0**、**Memu**、**MemOS**，均使用 GPT-4o-mini 构建记忆，用 DeepSeek-V3 生成回复。
  - **RAG 评估**：用四款嵌入模型检索后，均由 DeepSeek-V3 生成回复。
  - **评估指标**：采用 GPT-4o 作为裁判，从**流畅度、拟人度、信息量、共情度**四维度评分（1~5）。同时引入**记忆类型匹配率（TMR）**，评价模型选用的记忆类型是否符合需求-记忆类型映射原则。为保证可靠性，也进行了人工评估（Cohen's kappa 与 GPT-4o 评分达 0.77、0.75 的高一致性）。

## 6. 主要结论与发现

- **检索性能严重不足**：基于嵌入的语义相似度检索在 Top-1 准确率上全部低于 10%，最高 Recall@10 仅 46.42%，说明语义相似度远不足以捕获情感需求。
- **共情效果高度依赖记忆的恰当性**：使用不当记忆会严重拉低共情得分（如 DeepSeek-v3 在 base 设定共情 4.38，用黄金记忆则提升至 4.91），RAG 与 Agentic Memory 系统得分均显著低于 w/Golden 上限。
- **LLM 存在系统性偏差**：即使缩小检索空间，模型 (Qwen-Max, DeepSeek-v3 等) 的记忆选择普遍与需求不匹配，**爱与归属**需求常错误地选择高光记忆而非关系记忆，导致该维度共情得分最低。
- **思维链 (CoT) 提升有限**：CoT 推理能一定程度上改善 TMR 和部分共情分数，但与黄金记忆上限相比仍存在较大差距，表明代理选择情感恰当记忆的能力有根本性局限。
- **生成能力并非瓶颈**：即使在 w/Golden 设定下，部分模型在有效整合记忆并生成情感贴切回复上仍有提升空间。
