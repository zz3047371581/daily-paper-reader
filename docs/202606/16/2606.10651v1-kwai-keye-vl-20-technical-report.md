## 摘要
我们提出 Kwai Keye-VL-2.0-30B-A3B，一个开源的混合专家（MoE）多模态基础模型，旨在推进长视频理解和智能体智能。为应对小时级视频中超长上下文、信息冗余和极高计算成本的挑战，Keye-VL-2.0 首次将 DeepSeek 稀疏注意力（DSA）适配至基于 GQA 的多模态架构，在捕捉关键帧和长程时序依赖的同时，实现无损 256K 上下文处理。该架构由高度优化的训练和推理基础设施支撑，包括可扩展的视频 I/O、异构 ViT-LM 并行以及定制 DSA 内核，显著提升吞吐量并最小化计算开销。此外，为克服多任务对齐中的灾难性遗忘算法困境，我们引入跨模态多教师在线策略蒸馏（MOPD），并与 Context-RL 和 Video-RL 相结合。通过将在线策略生成的密集 token 级教师反馈蒸馏回仅激活 3B 参数的 MoE 骨干网络，Keye-VL-2.0 天生具备在代码、工具和搜索场景中的高级智能体协作能力，并支持多模态自我修正。在视频理解、时间定位、推理、STEM 和智能体基准上的广泛评估表明，Keye-VL-2.0-30B-A3B 在相似规模模型中达到最先进性能，尤其在 TimeLens 的细粒度时间定位和 Video-MME-v2 及 LongVideoBench 的长视频理解方面表现卓越。我们发布模型检查点，以加速社区向可扩展且鲁棒的多模态智能体应用迈进。

## Abstract
We introduce Kwai Keye-VL-2.0-30B-A3B, an open-source Mixture-of-Experts (MoE) multimodal foundation model designed to advance long-video understanding and agentic intelligence. To address the challenges of ultra-long contexts, information redundancy, and prohibitive computational costs inherent in hour-level videos, Keye-VL-2.0 is the first to adapt DeepSeek Sparse Attention (DSA) to GQA-based multimodal architectures, enabling lossless 256K context processing while capturing critical frames and long-range temporal dependencies. This architecture is underpinned by a highly optimized training and inference infrastructure, including scalable video I/O, heterogeneous ViT-LM parallelism, and custom DSA kernels that significantly maximize throughput and minimize computational overhead. Furthermore, to overcome the algorithmic dilemma of catastrophic forgetting during multi-task alignment, we introduce Cross-Modal Multi-Teacher On-Policy Distillation (MOPD) paired with Context-RL and Video-RL. By distilling dense token-level teacher feedback from on-policy rollouts back into the MoE backbone, which activates only 3B parameters, Keye-VL-2.0 natively empowers advanced agent collaboration across Code, Tool, and Search scenarios with multimodal self-correction. Extensive evaluations across video understanding, temporal grounding, reasoning, STEM, and agent benchmarks demonstrate that Keye-VL-2.0-30B-A3B achieves state-of-the-art performance among models of similar scale, particularly excelling in fine-grained temporal localization on TimeLens and long-video comprehension on Video-MME-v2 and LongVideoBench. We release our model checkpoints to accelerate community progress toward scalable and robust multimodal agentic applications.

### **2. 方法论：核心思想与关键技术细节**

论文的核心方法论围绕 **“极致上下文扩展”** 和 **“解决模态/任务冲突”** 两大主线展开。

**2.1 极致上下文扩展：多模态稀疏注意力与高分辨率编码**

-   **核心思想**：首次将专为大语言模型设计的**DeepSeek 稀疏注意力（DSA）** 成功适配到基于 GQA 的多模态架构中，通过稀疏化视频特征聚合，将计算复杂度从 O(L²) 降至 O(Lk)，实现线性成本下的长上下文处理。
-   **关键技术细节**:
    -   **原生分辨率视觉编码器**：采用自适应位置编码和二维旋转位置嵌入（2D RoPE），允许视觉编码器（ViT）直接处理原始尺寸和长宽比的图像与视频帧，保留全局结构和局部细节。该编码器在与下游 MLLM 任务相同的数据分布上进行了预训练。
    -   **统一的视觉编码策略**：将视频的每一帧视为独立高分辨率图像，用同一 ViT 编码。为每帧视觉特征前添加自然语言时间戳，作为语言模型（LLM）感知时序的锚点。同时采用自适应像素预算策略，根据视频长度动态分配计算资源。
    -   **GQA 兼容的 DSA**：
        1.  **MQA 风格闪电索引器（Lightning Indexer）**：基于多查询注意力（MQA）设计，所有查询头（Query Heads）共享一个键头（Key Head），高效地为每个查询 token 计算全局索引分数 `It,s`，并选出 Top-k 个 token 作为稀疏索引集 `Ωt`。
        2.  **GQA 稀疏聚合（Sparse Aggregation）**：将选出的相同稀疏索引集 `Ωt` 应用于所有查询头组，各组在其内部进行稀疏注意力计算，得到输出 `ut,g`。此步骤将稠密全局注意力替换为稀疏注意力，大幅缩减计算量。
        3.  **预热与适应训练**：
            -   **稠密预热阶段**：冻结主干网络，仅用 KL 散度损失 `LI_warmup` 训练索引器，使其输出的全局分布能覆盖所有 GQA 组的稠密注意力分布。
            -   **稀疏适应阶段**：解冻所有参数，切换至稀疏模式。索引器继续用 KL 散度 `LI_sparse` 在选中的 Top-k token 集合 `St` 上与主干模型对齐，模型主干则用标准的下一 token 预测损失 `LNTP` 进行优化。总损失为 `Ltotal = LNTP + λLI_sparse`。

**2.2 解决模态/任务冲突：跨模态多教师在线策略蒸馏与强化学习**

-   **核心思想**：通过**跨模态多教师在线策略蒸馏（MOPD）**，将多个特定领域的教师模型的专业能力，通过 token 级的在线反馈信号，无缝整合到统一的学生模型中，避免了端到端共优化带来的任务间干扰和灾难性遗忘。
-   **关键技术细节**:
    -   **MOPD 流程**:
        1.  **学生生成（On-Policy Rollout）**：学生模型针对给定提示 `xi`，生成在线策略响应 `yi`。
        2.  **教师路由与反馈**：根据样本的模态和任务类型，动态路由至对应的领域专家教师模型 `πT_r(i)`。教师模型接收学生生成过程中的状态 `si,t`，提供 token 级的对数概率反馈。
        3.  **带优势的策略梯度优化**：基于教师和学生对下一 token 的预测差异，计算一个原始优势值 `Ai,t`。该优势值仅在教师和学生的 Top-K 重叠集合 `Ωi,t` 内计算，确保训练稳定。学生模型通过一个优势加权的策略梯度目标进行优化：
            `LMOPD = -E[ (1/|Mi|) Σ bAi,t log πθ(yi,t | xi, yi,<t) ]`
            其中 `bAi,t` 是经过缩放和去重惩罚后的最终优势值。
    -   **强化学习（RL）系列**：
        -   **通用和专用 RL**：采用**分组序列策略优化（GSPO）** 算法，在数学、OCR、定位等可验证任务上使用基于规则或模型裁判的奖励进行优化。
        -   **上下文 RL（Context-RL）**：将模型生成的响应与已验证的参考答案进行对比，提供更细粒度的过程奖励，减少因错误推理得出正确答案的误判。
        -   **视频 RL**：采用时间交并比（tIoU）和 LLM-as-Judge 等奖励，专门优化时间定位和密集视频描述。
        -   **智能体 RL**：在代码、工具使用和搜索环境中，使用由环境执行结果决定的轨迹级奖励进行优化。

-   **资源和成本细节缺失**：报告未提供训练模型所需的总算力、GPU 数量和具体训练时长等关键信息，使得其他研究人员难以精确复现或评估其资源门槛。

-   **长文本能力评估不均衡**：虽然重点评估了长视频理解，但对超长文本（如长文档、长代码）的理解能力仅有定性描述，缺乏定量基准测试对比。
-   **智能体评估场景简化**：智能体任务评估集中于代码、工具调用和搜索，属于比较“干净”的交互环境。对于更复杂、开放的真实世界物理环境或社交交互场景下的智能体能力尚未涉及。
-   **幻觉问题未量化评估**：虽然在方法上提到了通过 RL 减少视觉幻觉，但仅在部分案例中提及，缺乏在全量评测集上的具体量化改善数据。
