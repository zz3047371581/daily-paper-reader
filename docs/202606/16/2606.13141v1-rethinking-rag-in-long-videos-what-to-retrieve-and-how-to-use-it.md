## 摘要
检索增强生成正从文本领域扩展到长时间、以自我为中心的视频，系统必须在多种模态和时间粒度中选择与查询相关的片段。然而，VideoRAG的进展受限于两个不足：现有的基准测试允许在没有视频的情况下回答查询，这掩盖了检索错误；而先前的方法对每个查询只应用单一的模态-粒度配置，忽略了片段级别的变化。我们通过引入V-RAGBench和CARVE来解决这两个问题。V-RAGBench是一个由⟨查询、证据片段、答案⟩三元组构成的基准测试，能够对检索和生成进行忠实、解耦的评估；CARVE是一种简单的方法，它在不同的配置下并行运行检索器，并采用片段自适应重排序来确定每个片段的最优配置。然后，每个片段以其在检索阶段选择的最优配置进入生成器，产生一种交错证据形式，其中片段级别的决策在两个阶段中传播。CARVE优于最近的八种VideoRAG基线方法，提供给生成器的片段交错采用了多种配置，而不是共享单一配置，这是查询级方法无法实现的行为。

## Abstract
Retrieval-augmented generation is moving beyond text into long, egocentric video, where systems must select query-relevant chunks across multiple modalities and temporal granularities. Yet progress in VideoRAG is limited by two gaps: existing benchmarks allow queries to be answered without the video, obscuring retrieval errors, and prior methods apply a single modality-granularity configuration per query, ignoring chunk-level variability. We address both by introducing V-RAGBench, a benchmark of $\langle$query, evidence chunk, answer$\rangle$ triplets that enables faithful, decoupled evaluation of retrieval and generation, and CARVE, a simple method that runs parallel retrievers across configurations and employs chunk-adaptive reranking to identify the winning configuration for each chunk. Each chunk then enters the generator under its winning configuration selected during retrieval, yielding an interleaved evidence form where the chunk-level decision propagates across both stages. CARVE outperforms eight recent VideoRAG baselines, with the chunks supplied to the generator interleaving multiple configurations rather than sharing a single one, a behavior unattainable by query-level methods.

### 2. 论文提出的方法论：核心思想、关键技术细节

论文提出了一种名为**CARVE（Chunk-AwaRe Reranking for Video Evidence）**的方法，以及支撑其评测的**V-RAGBench**基准。

#### V-RAGBench基准构建
- 从Ego4D和EgoLife中筛选216段时长大于1小时的自我中心视频。
- 经过事件提取和语义去重，利用大模型生成候选QA对。
- 通过五重后处理过滤器（语义相似度、可回答性、捷径偏差、经验可回答性、证据唯一性）确保每个查询严格依赖唯一的视频证据片段。
- 最终形成包含2100个高质量⟨查询, 证据片段, 答案⟩三元组的平衡数据集。

#### CARVE方法框架
- **视频片段表示**：将视频切分为2分钟非重叠片段，每个片段交叉产生四种候选配置：
  - 模态轴：视觉（像素级特征）和文本（MLLM生成的文字描述）
  - 粒度轴：帧级别（关键帧细节）和片段级别（整体摘要）
- **阶段一：并行候选池化**：针对查询，在每个配置下独立检索Top-k片段，汇聚到一个带配置标签的候选池中。
- **阶段二：片段自适应重排序**：使用多模态交叉编码器对每个候选片段仅按其所带的配置标签评分，保留每个片段的最高得分配置作为“获胜配置”。
- **模态交织生成**：将最终选出的Top-k片段以各自的获胜配置（如文本-帧、视觉-片段等）混合送入MLLM生成器，形成片段级配置交错的证据上下文。

### 6. 论文的主要结论与发现

- **基准层面**：V-RAGBench通过严格的三重属性（非重复证据、视觉依赖、唯一证据定位）实现了检索与生成的解耦评价，能暴露真正的检索弱点。
- **方法层面**：
  - CARVE在检索和生成两个阶段均显著优于所有系统级和查询级基线（Recall@5达0.603，比最佳基线高9个百分点以上）。
  - 片段级配置选择不是单一配置的简单放大，四种配置在不同片段上均匀涌现，交替使用能产生协同增益。
  - 在生成阶段，CARVE采用交错证据形式，其性能甚至超越经过训练来选择单一配置的查询级路由器。
  - 混合模态比混合粒度更可靠地带来提升；CARVE的延迟适中，因为只需对每个片段传递其获胜配置，避免了昂贵的全量视觉重排。
- **总体启示**：长视频RAG应当将“用什么表示检索及生成”的决策下沉到片段级别，并让该决策在检索与生成阶段连贯传递。

### 7. 优点：方法或实验设计亮点

- **问题定位精准**：同时揭露基准缺陷与方法粒度缺失，填补VideoRAG空白。
- **基准构建严谨**：通过事件去重、多级过滤确保每个查询具备唯一且必需的视频证据，评测结果忠实可信。
- **方法创新简单有效**：巧妙地将交叉编码器重排机制转化为片段级配置选择策略，无需额外训练，工程成本低。
- **设计优雅**：并行检索->带标签重排->交错生成的流水线清晰，易于扩展。
- **评估全面**：涵盖检索和生成多指标、多个生成器、跨数据集分析、消融与鲁棒性测试，结论说服力强。

### 8. 不足与局限

- **模态与粒度设定有限**：仅探索了视觉/文本×帧/片段四种配置，未扩展到音频、OCR、时间戳等其他可能辅助检索的信号。
- **检索范式固定**：采用一次性检索-生成管道，未探索迭代式、代理式多轮检索场景，而后者在当下亦为主流方向。
- **领域局限**：基准和实验均构建于自我中心视频，尚未在第三人称视频（如电影、讲座）上验证泛化性。
- **片段时长固定**：默认2分钟块，虽然适配多数日常内容，但缺乏对动态可变长度片段的讨论。
- **重排器开销**：尽管已优化，视觉重排仍带来额外延迟；未来可研究更轻量化的跨模态打分方式。
- **小型微调开销**：用于对比的查询路由器需在V-RAGBench训练集上训练，且仅限研究目的，实际新部署时CARVE的零训练优势更明显。
