## 摘要
流式视频理解模型必须在持续流中的任何时刻回答查询，仅使用截至当时所观测到的内容，并受限于固定的内存和计算预算。现有方法通过添加记忆库、检索模块或视觉令牌压缩来保留长期历史。然而，强大的近期窗口基线显示，不加区分的历史注入会削弱当前场景的感知，这表明关键挑战不在于是否使用记忆，而在于如何选择性地分配记忆。我们将此问题形式化为有预算的在线潜在证据分配，并提出\textbf{SelectStream}，一种选择性潜在记忆框架，该框架将当前观测直接呈现给冻结的视觉语言模型（VLM），同时仅通过紧凑的、以查询为条件的证据预算来暴露历史信息。三个协调机制控制何时写入、保留什么以及如何检索：基于惊奇的适应性窗口、优先级保持的整合，以及在固定容量的潜在记忆图上进行的以查询为条件的图推理。检索到的证据经过校准后作为潜在令牌注入，以生成答案，无需重放帧或随流长度增加上下文。实验结果表明，SelectStream实现了强大的在线流式性能，并保持了通用视频理解能力，在StreamingBench上达到82.67%，在OVO-Bench上达到67.03%，在离线视频基准上的平均准确率为74.4%，同时优于强大的近期窗口基线和先前的流式记忆方法。

## Abstract
Streaming video understanding models must answer queries at any moment during an ongoing stream, using only what they have observed so far and under fixed memory and computation budgets. Existing methods address this by adding memory banks, retrieval modules, or visual token compression to preserve long-range history. However, strong recent-window baselines show that indiscriminate history injection can dilute current-scene perception, suggesting that the key challenge is not whether to use memory, but how to allocate it selectively. We formulate this as budgeted online latent evidence allocation and propose \textbf{SelectStream}, a selective latent-memory framework that keeps the current observation directly visible to a frozen VLM while exposing historical information only through a compact, query-conditioned evidence budget. Three coordinated mechanisms govern when to write, what to preserve, and how to retrieve: surprise-driven adaptive windowing, priority-preserving consolidation, and query-conditioned graph reasoning over a fixed-capacity latent memory graph. Retrieved evidence is calibrated and injected as latent tokens for answer generation, without replaying frames or growing the context with stream length. Experimental results show that SelectStream achieves strong online streaming performance and preserves general video understanding, reaching 82.67\% on StreamingBench, 67.03\% on OVO-Bench, and 74.4\% average accuracy on offline video benchmarks, while outperforming strong recent-window baselines and prior streaming memory methods.

### 1. 论文的核心问题与整体含义

*   **研究动机**: 流式视频理解（如自动驾驶、AR助手）对模型提出了独特要求：在任何时刻基于已观察的、有限的过去信息回答查询，且受固定内存和计算预算限制。
*   **核心矛盾**: 现有方法为保留长期历史，盲目添加记忆库或压缩历史帧，但近期研究表明，不加区分地注入过多历史信息会“稀释”对当前场景的感知，反而降低性能。
*   **根本问题**: 挑战不在于“是否使用记忆”，而在于“如何在固定预算下，**选择性地分配记忆**”，即决定何时写入新证据、容量不足时保留什么、以及如何为给定查询检索最相关的历史证据。

### 2. 论文提出的方法论

*   **核心思想**: 将流式视频记忆管理形式化为**预算在线潜在证据分配**问题，提出 **SelectStream** 框架。它在冻结的视觉语言模型（VLM）之上，通过三个协调的模块，将历史信息组织成一个动态潜在证据图，仅以紧凑的、经查询校准的潜在令牌形式暴露给模型。
*   **关键技术细节**:
    *   **惊喜驱动自适应窗口（SAW）**：决定“何时写入”。通过计算相邻帧间的注意力偏移和特征变化来估计“惊喜度”，在惊喜度峰值或累积能量超过阈值时关闭一个片段，从而在稳定期使用粗粒度记忆，在事件转折点保留精细记忆。
    *   **潜在视觉记忆（LVM）**：决定“保留什么”。将SAW切分的片段编码为潜在状态节点，构成一个动态记忆图。当记忆容量超过预算时，通过**优先级保持的整合**，优先合并在语义上相似且具有低惊喜度、低访问频率、更早更新的节点，保护重要证据不被丢弃。
    *   **查询条件图推理（GAR）**：决定“如何读取”。当查询到达时，首先通过粗粒度评分选出与查询语义相关、惊喜度高的少量种子节点，然后以查询为条件在图上进行路由，构建一个不超过预算的子图。在此子图上执行关系图注意力推理，最终重排序并选出Top-M个节点。
    *   **潜在证据注入**: 选出的M个节点状态，通过一个可训练的校准层（LayerNorm + 线性映射）转换为“校准潜在证据令牌”，与当前帧和用户提示一同输入冻结的VLM解码器生成答案，无需重放原始帧或文本总结。

*   **性能提升显著**: SelectStream在所有在线流式基准上均取得最佳结果，不仅在需要长期回溯的`Backward Tracing`任务上大幅领先（+9%以上），还保持了顶级的实时感知能力，验证了选择性记忆能有效平衡历史与当下。
*   **泛化能力不降反升**: 在离线视频基准上，SelectStream相比原始的冻结VLM平均准确率还有提升（如70.4% vs. 68.3%），说明这种记忆机制并未牺牲通用理解能力，反而有助于长视频理解。
*   **效率可控**: 通过控制N、B、M三个解耦的预算参数，可以灵活权衡精度与延迟/显存。模型在处理的流长度增加时，延迟和显存几乎不变，证明了其O(1)的查询时间复杂性。
*   **结论**: 流式视频模型成功的关键不在于记忆更多，而在于更聪明地管理记忆。**预算化、选择性的潜在证据分配**是解决这一问题的有效途径。

### 7. 优点

*   **问题定义新颖且合理**: 将宏观的记忆挑战具体化为“写、留、读”的联合预算分配问题，直击痛点。
*   **设计紧致优雅**: 整个框架由三个功能明确、高度协同的模块构成，且所有历史交互都在紧凑的潜在空间进行，避免了高成本的回放或文本转换。
*   **实验扎实且有说服力**: 对比基线全面（尤其是与近期窗口基线的对比），消融和分析实验深刻，有效支撑了设计选择。
*   **效率与准确性兼顾**: 通过解耦的预算参数设计，实现了性能与资源消耗的可控平衡，具有良好的部署前景。

### 8. 不足与局限

*   **模型规模探索有限**: 仅在7-8B参数规模的VLM上验证，未展示该方法在更大规模（如14B、32B、72B）或不同类型VLM上的可扩展性。
*   **固定预算依赖人工设定**: 内存容量N等关键预算参数目前是固定的人为设定的，无法根据流内容的复杂度动态自适应调整。
*   **“惊喜度”信号的局限性**: 惊喜度计算依赖于注意力图或特征变化，在某些场景变化缓慢但语义持续演进的长视频中（如纪录片、教学视频），可能无法有效触发记忆写入。
*   **离线数据的因果性偏差**: 虽然离线基准评估使用了“因果最终查询协议”，但训练数据`Streamo-Instruct`源于处理后的离线视频，其数据分布可能与真实永无止境的实时流存在偏差。
