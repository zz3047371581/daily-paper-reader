## 摘要
近期的大型多模态模型主要针对离线场景设计，难以满足流式视频的动态需求。尽管已有在线适配方法提升了实时处理能力，但仍面临严峻的可扩展性挑战，其资源需求通常至少随视频时长线性增长。为克服这一瓶颈，我们提出 FlowNar，一种面向可扩展流式视频解说的新型框架。其核心是一种用于历史视觉上下文移除的动态上下文管理策略，结合我们提出的 CLAM（交叉线性注意力记忆）模块实现流式视觉历史的保留，从而确保有界的视觉内存使用和计算复杂度，这对高效流式处理至关重要。我们还引入了一种现实的自我条件化评估协议及互补评估指标，以在接近部署环境下评估流式解说模型。在 Ego4D、EgoExo4D 和 EpicKitchens100 数据集上的实验表明，FlowNar 在显著提升解说质量的同时保持高效，支持处理长达 10 倍时长的视频，并实现 3 倍的吞吐量（FPS）。代码已开源：https://github.com/zeyun-zhong/FlowNar。

## Abstract
Recent Large Multimodal Models (LMMs), primarily designed for offline settings, are ill-suited for the dynamic requirements of streaming video. While recent online adaptations improve real-time processing, they still face critical scalability challenges, with resource demands typically growing at least linearly with video duration. To overcome this bottleneck, we propose FlowNar, a novel framework for scalable streaming video narration. The core of FlowNar is a dynamic context management strategy for historical visual context removal, combined with our CLAM (Cross Linear Attentive Memory) module for streaming visual history retention, ensuring bounded visual memory usage and computational complexity, crucial for efficient streaming. We also introduce a realistic self-conditioned evaluation protocol and complementary evaluation metrics to assess streaming narration models under deployment-like conditions. Experiments on the Ego4D, EgoExo4D, and EpicKitchens100 datasets demonstrate that FlowNar substantially improves narration quality over strong baselines while being highly efficient, supporting processing of 10$\times$ longer videos and achieving 3$\times$ higher throughput (FPS). The code is available at https://github.com/zeyun-zhong/FlowNar.

### 1. 核心问题与研究动机
- 现有大型多模态模型（LMMs）多为离线设计，难以满足流式视频的实时、连续解读需求。
- 已有在线流式解说方法（如 Videollm-online）虽可逐帧生成描述，但其视觉上下文（Key-Value cache）随视频时长不断累积，导致显存占用和计算量至少线性增长，极易超出 GPU 显存上限或显著降低推理速度。
- 如何实现资源开销可控、不随时间无限增长的长时视频流式解说，是该领域亟需突破的核心瓶颈。

### 2. 方法论
#### 2.1 总体框架：FlowNar
- 将无剪裁长视频视为帧序列，在线地、自适应地决定“何时解说”与“解说内容”。
- 生成过程以递归方式行进，每段解说 \(y_n\) 均以动态维护的视觉上下文 \(C^{vid}_t\) 与解说历史 \(C^{nar}_n\) 为条件。

#### 2.2 动态上下文管理（DCM）
- **核心原则**：每完成一段解说后，显式丢弃该段内原始帧特征的 KV 缓存，阻止视觉上下文无限增长。
- **效果**：将视觉上下文复杂度保持在常数级，同时减少因依赖过去可能存在错误的解说历史而产生的误差传播。

#### 2.3 交叉线性注意力记忆（CLAM）
- 为解决简单去除视觉历史导致的长期信息丢失，提出一种可增量压缩过去视觉信息的模块。
- **关键特性**：恒定的内存占用量、每一步恒定的计算复杂度、训练时可高度并行化。
- **机制**：
  - 维持一个 \(D \times D\) 的循环状态矩阵 \(S_t\)，通过门控递推依次处理每帧的 token：
    \[ S_{t,j} = G_{t,j} \odot S_{t,j-1} + k_{t,j}^\top v_{t,j} \]
    其中 \(G_{t,j}\) 为逐元素衰减门，控制历史保留程度。
  - 利用可学习的查询向量 \(Z\) 经过线性变换得到查询矩阵 \(Q\)，从最终状态 \(S_t\) 读出一组固定数量的记忆 token：\(M_t = Q S_t\)。
- 记忆 token 在下个片段的起始处被前置给帧 embedding，作为对远过去视觉信息的压缩摘要。

#### 2.4 解说触发与生成
- 使用大语言模型同时判定触发时机和文本生成。
- 每帧判断特殊令牌 `[SKIP]` 的概率是否低于阈值 \(\theta\) 来决定解说终点 \(t_n\)。
- 为避免频繁连续触发或长时间沉默，采用**动态双阈值策略**：默认使用主阈值 \(\theta\)，刚生成解说后短暂切换至更低阈值 \(\theta_{low}\) 抑制快速触发。
- 生成解说后，更新解说上下文 \(C^{nar}_n\)；视觉缓存被清空，新的记忆 token \(M_{t_n}\) 作为下一段的历史摘要。

#### 2.5 FlowNar-C 变体
- 在 DCM 基础上，进一步对文本解说上下文应用**最近 k 条保留策略**，使解说历史的 KV 缓存也保持恒定，实现全维度有界内存。

#### 2.6 训练策略
- 在含多个连续片段的长视频上训练。
- 采用专门的**注意力掩码**：在标准因果掩码之外，屏蔽对远过去片段的原始视觉 token 和旧记忆 token 的直接注意力，迫使模型只依赖当前片段和最近的记忆 token。
- 使用标准交叉熵损失联合预测解说词元和触发用的 `[SKIP]` token。
- 推理时通过维护位置计数器模拟训练时的 positional ID，解决因果缓存操作造成的 ID 不一致问题。

### 3. 实验设计
#### 3.1 数据集
- **Ego4D** (10万+ 训练集)、**EgoExo4D** (约3200 训练集)、**Epic-Kitchens100 (EK100)** (495 训练集)：均为长时第一人称视频，包含时间戳叙事标注。

#### 3.2 评估协议与指标
- **效率**：MACs (乘加运算数) 与峰值 GPU 显存 (VRAM)。
- **教师强制协议 (teacher-forcing)**：以真实历史解说完备模型输入，报告 PPL、LM-Correctness、TimeDiff、Fluency 等（用于与文献对标）。
- **自我条件协议 (self-conditioned)**：模型仅依赖自身先前生成的解说，更贴近真实部署。采用“先对齐再评估”策略：先将预测片段与真实片段按 IoU 匹配，再计算：
  - 时间对齐：精确率、召回率、F1 (IoU≥0.5)
  - 解说质量：CIDEr、METEOR、ROUGE-L（基于 GIoU 最佳匹配）

#### 3.3 对比方法
- 主要基线：**Videollm-online**、**Videollm-mod**、**LION-FS**（后者仅教师强制协议下比较）。
- 所有模型统一使用 SigLIP-L/16 视觉编码器和 Llama-3（1B/8B）语言模型。

### 6. 主要结论与发现
- FlowNar 通过 DCM+CLAM 实现了视觉上下文的有界化，显著降低显存和计算开销，支持长时间流式处理。
- 在自条件评估下，FlowNar 在三个数据集上均较大幅度超越 Videollm-online 等强基线，尤其在召回率和叙述质量上提升明显。
- CLAM 作为一种定容循环记忆，相比于简单丢弃历史或仅保留最近帧，能更有效地保留长期视觉信息，且优于其他在线压缩方案。
- 动态双阈值触发策略有效改善了叙述节奏，避免沉默或突发触发，进一步提升整体性能。
- 在零样本切换到第三人称视频时，模型仍能捕捉核心语义，但叙述风格偏向第一人称训练数据。

### 7. 优点
- **可扩展性突破**：首次在流式视频解说中实现视觉内存的恒定复杂度，支持数倍于以往方法的长视频处理。
- **记忆模块设计精巧**：基于线性注意力的循环门控状态与可学习查询读出，兼顾了信息压缩与任务适配。
- **贴近现实的评估**：提出的自条件评估协议和“先对齐再评估”流程更真实地反映部署场景下的性能，避免教师强制评估的伪装。
- **实验全面**：多个数据集、多个基线、丰富消融与可解释性分析，结论可信。

### 8. 不足与局限
- **训练开销相对较高**：因额外的记忆 token 及片段级注意力掩码导致训练耗时为基线的 1.9 倍，该部分被视为可优化的工程瓶颈。
- **评估范围集中于第一人称叙事**：尽管在零样本下展示了向第三人称泛化的潜质，但定量评估仅限于三个第一人称数据集。
- **对极端视觉条件的敏感性**：在低光照等情形下可能漏检或产生对象幻觉，可能与激进的空间压缩（3×3 池化）有关。
- **文本记忆的适用性有限**：实验表明 CLAM 不适合直接压缩解说文本历史，仍需采用简单的最远 k 句策略。
