## 摘要
CoVR-R研究推理感知的组合视频检索：给定一个参考视频和一个编辑指令，系统必须检索出满足该编辑的目标视频。主要困难在于目标视频并未直接描述，必须从物体身份、动作顺序、最终状态、手部交互以及场景转换的细粒度变化中推断出来。我们基于Qwen3.5-27B构建了一个零样本的先推理后检索流程。对于每个库中视频，模型生成一个面向检索的结构化描述，并通过使用令牌相关权重对生成的令牌隐藏状态进行池化来获得稠密嵌入。对于每个查询，模型首先对参考视频和指令进行编辑推理，然后生成目标视频描述，其隐藏状态作为查询嵌入。我们通过一个对生成文本的TF-IDF分支来补充稠密检索，并用分裂特定的权重融合两个排序结果。在验证集上，当前最佳提交在R@1达到80.81，R@5为94.86，R@10为97.11，R@50为98.59。在盲测集上，它在R@1达到89.73，R@5为95.79，R@10为96.63，R@50为97.98。

## Abstract
CoVR-R studies reason-aware composed video retrieval: given a reference video and an edit instruction, the system must retrieve the target video that satisfies the edit. The main difficulty is that the target is not described directly; it must be inferred from fine-grained changes in object identity, action order, final state, hand interaction, and scene transition. We build a zero-shot reason-then-retrieve pipeline around Qwen3.5-27B. For each gallery video, the model generates a retrieval-oriented structured description and a dense embedding by pooling generated-token hidden states with token-dependent weights. For each query, the model first performs edit reasoning over the reference video and instruction, then generates a target-video description whose hidden states serve as the query embedding. We complement dense retrieval with a TF-IDF branch over the generated texts and fuse the two rankings with split-specific weights. On validation, the current best submission reaches 80.81 at R@1, 94.86 at R@5, 97.11 at R@10, and 98.59 at R@50. On the blind test split, it reaches 89.73 at R@1, 95.79 at R@5, 96.63 at R@10, and 97.98 at R@50.

## 方法论
- **总体策略**：零样本“先推理后检索”（reason-then-retrieve）。不进行任务特定微调，所有适配来自提示设计、令牌加权池化和稠密-稀疏融合。
- **主干模型**：使用 Qwen3.5-27B 作为视觉语言核心。
- **库视频表示**：
  - 为每个库视频生成一段“面向检索的结构化描述”，提示要求保留主要物体、属性、动作链、手部交互、最终状态、场景转换等信息。
  - 将生成的描述通过令牌权重池化获得稠密嵌入：`g = Σ(w_i * h_i) / Σ(w_i)`，其中 `h_i` 为生成令牌 i 的隐藏状态，`w_i` 为重要性权重（内容词赋予更大权重，停止词、标签等小权重），池化后 L2 归一化。
- **查询表示（编辑推理）**：
  - 先执行编辑推理：判断哪些内容应保留、哪些应消失、哪些新动作或状态应出现。
  - 再基于参考视频、指令和推理轨迹生成目标视频的结构化描述。
  - 以相同令牌权重池化方式从生成的目标描述中提取查询嵌入。
- **稠密检索**：计算查询嵌入与库嵌入的余弦相似度 `s_dense = (q·g)/(||q||·||g||)`。
- **稀疏检索与融合**：
  - 额外构建 TF-IDF 分支（基于生成的文本描述），利用 scikit-learn 实现。
  - 融合得分：`s_final = α·ŝ_dense + (1-α)·ŝ_tfidf`，α 为分片特定权重（WebVid 与 Something-Something V2 采用不同混合比例，SS2 更侧重稠密分支）。
- **整体流水线**：库描述生成与嵌入提取 → 查询端推理与目标描述生成 → 稠密检索 → TF-IDF 稀疏检索 → 分数融合排序。

## 主要结论与发现
- 结构化提示显著提升表示质量：保留动作顺序、最终状态和过渡线索是检索有效的关键，尤其在 SS2 分片上。
- 稠密和稀疏检索分支高度互补：稠密分支处理语义改写，TF-IDF 分支保留精确物体、状态等词汇；融合后性能大幅优于单一分支。
- 零样本的“先推理后检索”管线和生成式文本表示，能够较好应对 CoVR-R 的推理需求，在盲测集取得 R@1=89.73% 的强劲结果，证明大模型直接通过提示即可输出高质量的编辑推理和目标描述。

## 优点
- **零样本方案**：无需额外微调，仅靠提示工程和表示池化，实用性强。
- **表示对齐**：通过生成检索导向的结构化描述，使库和查询表示与检索目标高度对齐，而非依赖通用摘要。

- **融合策略**：稠密-稀疏加权融合，配合分片特异的平衡系数，简单有效。
