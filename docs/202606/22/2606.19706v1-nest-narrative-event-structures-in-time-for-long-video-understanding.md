## 摘要
视觉-语言模型的最新进展使得处理越来越长的视频序列成为可能，但处理扩展令牌流的能力并不等同于理解长视频中的叙事结构。现有的长视频基准测试侧重于大海捞针式的检索，而非评估底层动作如何构成事件、事件如何跨时间交互以及叙事如何推进，例如，模型能否将早期的挫折（如失业）与后来的关系破裂联系起来，尽管存在长时间间隔、穿插场景或重塑所发生事件性质的闪回。我们引入了 NEST（面向长视频理解的时间叙事事件结构），这是一个包含 1005 部全长电影（平均片长 98 分钟）的数据集，每部电影标注了 102 个以视觉内容、对话和音频为基础的多模态叙事事件。NEST 通过结构化的标注捕捉多模态叙事事件，这些标注基于视觉内容、对话和音频，并通过反映叙事结构的关系（包括时序顺序、层次组合和长距离依赖）将它们联系起来。我们为事件触发检测（ETD）、事件定位（EL）、事件论元抽取（EAE）和事件关系抽取（ERE）设立了基线。该基准测试对于基于内容的底层事件发现极具挑战性，ETD 低于 8%，EL 低于 6%，EAE 低于 11%。相比之下，一旦给出事件，ERE 则更易处理，零样本 F1 值达到 35.45%，微调后达到 44.42%。

## Abstract
Recent progress in vision-language models has enabled the processing of increasingly long video sequences, but the ability to handle extended token streams does not translate to understanding of narrative structure in long videos. Existing long video benchmarks focus on needle-in-a-haystack retrieval rather than evaluating how low-level actions form events, how events interact across time, and how narratives progress, for example, whether a model can connect an early setback, such as a job loss to a later relationship breakup, despite long gaps, intervening scenes, or flashbacks that reframe what occurred. We introduce NEST (Narrative Event Structures in Time for Long Video Understanding), a dataset of 1005 full-length movies (avg. 98 minutes), each annotated with 102 multimodal narrative events grounded in visual content, dialogue, and audio. NEST captures multimodal narrative events with structured annotations grounded in visual content, dialogue, and audio, and links them through relations that reflect narrative structure, including temporal ordering, hierarchical composition, and long-range dependencies. We introduce baselines for event trigger detection (ETD), event localization (EL), event argument extraction (EAE), and event relation extraction (ERE). The benchmark is highly challenging for grounded event discovery, with ETD below 8%, EL under 6%, and EAE below 11%. In contrast, ERE is more tractable once events are given, reaching 35.45% F1 zero-shot and 44.42% F1 after fine-tuning.

## 2. 论文提出的方法论

### 2.1 数据集构建：NEST 的整体管线
- **数据源**：1005 部全长电影（平均 98 分钟），来源于 IMDb、YouTube、Library of Congress 等。
- **多模态信息融合**：
  - **视觉**：完整视频帧
  - **对话**：字幕转录
  - **音频**：音频描述（Audio Description，AD）——由专业解说员录制，忠实描述屏幕上的视觉内容，被用作事件提取的“金标准”来源。
- **处理步骤**：
  1. **场景分割**：使用 PySceneDetect 按自然过渡切割场景（平均每部 ∼160 个场景）。
  2. **指代消解**：用 Maverick 和 LLM 对音频描述中的人物进行跨场景一致实体 ID 分配。
  3. **多模态事件抽取**：基于 PropBank 约束的动词词表，结合 OmniEvent 等监督模型与 LLM，提取事件触发词和论元。
  4. **事件关系抽取**：识别事件间的**时序、因果、前提条件、层级、共指**等关系。
  5. **时间定位**：将事件对齐到视频场景时间戳（基于场景级评估，因精确时间边界仍具主观性和技术难度）。
- **标注规模**：
  - **SILVER 集**：1005 部电影的 LLM 辅助自动标注（消耗约 1000 亿 tokens）。
  - **GOLD 集**：5 部电影的高质量人工标注（5 名经过培训的标注员，报酬 $15/小时，总成本 $600）。

### 2.2 四项核心基准任务
1. **事件触发检测**（ETD）：给定视频与场景边界，模型需输出叙事级动词及上下文描述（非表面物理动作）。
2. **事件定位**（EL）：给定事件描述，模型需预测其在长视频中的时间边界（场景级评估）。
3. **事件论元抽取**（EAE）：给定触发词和语义角色，模型需填空参与者和情境细节（如施事者、受事者）。
4. **事件关系抽取**（ERE）：给定一对事件，预测其关系类型（时序/因果/前提条件/层级/共指/无关系），并专门评估**闪回子集**上的非线性时序推理。

### 2.3 技术细节与评估指标
- **事件本体**：严格基于 PropBank 语义角色框架（ARG0 施事、ARG1 受事、ARGM 修饰语等）。
- **ETD/EAE 评估**：由于开放式输出的同义性，采用 **LLM-as-judge**（GPT-5-mini）进行语义等价判断。
- **EL 评估**：检查预测时间戳是否落入真实场景边界内（“场景级重叠”作为低主观性代理）。
- **ERE 评估**：直接与真实关系标签对比计算 F1。

## 6. 主要结论与发现

### 发现一：**基于内容的叙事事件发现仍未解决**
- **ETD < 8%、EL < 6%、EAE < 11%**：所有模型在从原始视频中识别“发生了什么、谁做的、何时何地”等底层事件结构上表现极差。
- **增加帧数无帮助**：瓶颈在于叙事抽象能力而非视觉覆盖不足。

### 发现二：**非线性时序推理灾难性薄弱**
- **闪回子集上 6/7 模型 F1 = 0.00**，仅 Qwen2.5-VL（32B）得分 28.57%：模型严重依赖隐式的线性时间线假设，无法处理记忆回放、倒叙等常见叙事装置。

### 发现三：**论元抽取瓶颈是跨长片实体追踪**
- EAE 90% 的错误来自实体混淆（角色张冠李戴），仅 10% 为缺失论元：模型试图填空，但缺乏跨小时的视觉重识别和指代链维护能力。

### 发现四：**事件关系提取（给定事件后）相对可行，但短板仍在**
- 零样本 ERE 最佳 F1 达 35.45%，微调后提升至 44.42%。
- 共指和因果关系较易学习；时序、前提条件和层级关系仍困难。
- **模型在定位时过于自信但出错**：EL 中 98.6% 错误为预测特定错误场景（极少输出宽泛区间）。
- **在关系提取上过于保守**：ERE 中 78.3% 错误为将真实关联误判为 NO_RELATION，漏掉大量时序和因果链接。

### 微调的局限
微调使 ERE 从 35.45% 提升至 44.42%，但对于 ETD 和 EL 的提升极为有限（ETD 3.20% → 6.09%，EL 0.44% → 0.45%），说明**纯 SFT 不足以获取长程叙事归纳与定位能力**。

- **场景级定位的粒度不足**：用场景边界代替精确时间戳评估 EL，虽降低主观性，但忽略了对细粒度时序理解的测试（作者已承认，并期待随技术成熟迭代）。
- **跨场景复合事件缺失**：当前标注局限于单场景内事件，未能刻画在多场景间展开的渐进性事件（如人物性格弧光）。
- **LLM-as-judge 内在偏差**：ETD 和 EAE 依赖 GPT-5-mini 进行语义判断，可能引入评判模型自身的局限性。
- **标注资源差异**：GOLD 集仅 5 部电影，人工评估范围有限；SILVER 集虽大但为 LLM 辅助生成，存在模型偏差风险。
- **计算成本透明度**：未报告微调所需的总 GPU 小时数；仅提及训练硬件和策略，较难直接评估复现的全栈算力门槛。
- **版权限制**：仅能分发特征而非全部原始视频，限制了某些端到端训练/分析形式。
